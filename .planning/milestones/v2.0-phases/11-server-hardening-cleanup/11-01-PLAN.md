   Same reason as above.

   `res.setHeader('Access-Control-Allow-Methods', 'GET, POST, OPTIONS');`
   to
   `res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PATCH, DELETE, OPTIONS');`

   `await supabase.from('dj_availability').delete().eq('month', month);`
   to

   if (availDelError) throw new Error(availDelError.message);

   This follows the same error-check pattern already used for the dj_submissions and roster_assignments deletes on lines 1261-1266.
