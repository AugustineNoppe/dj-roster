    Modal title: "Recurring Availability: {dj.name}"

    Build a checkbox grid table:
    - **Columns:** Sun, Mon, Tue, Wed, Thu, Fri, Sat (day-of-week 0-6)
    - **Rows:** Use a unified slot list. Combine all unique slots from `ARKBAR_SLOTS` (which is the largest set, 11 slots covering 14:00-02:00). This covers all possible recurring availability slots.
    - **Each cell:** A checkbox. `id="avail-{dow}-{slotIndex}"`.
    - **Pre-load:** For each day `dow` (0-6), check if `dj.recurring_availability` has that key. If so, for each slot in the row, check if the slot string is in the array `dj.recurring_availability[String(dow)]`. If yes, set checkbox to checked.
    - Handle null `recurring_availability` gracefully (all unchecked).

    Add CSS for the grid:
    ```css
    .avail-grid{border-collapse:collapse;width:100%;font-size:12px}
    .avail-grid th{padding:4px 6px;text-align:center;color:var(--text-secondary);font-size:11px}
    .avail-grid td{padding:4px 6px;text-align:center}
    .avail-grid input[type="checkbox"]{width:16px;height:16px;cursor:pointer}
    ```

    Buttons: "Save" and "Cancel".

    **2. `saveAvailability(djId)` function:**
    - Iterate over days 0-6 and all slot checkboxes
    - For each day, collect checked slot strings into an array
    - Build `recurring_availability` object: `{ "0": [...], "1": [...], ... }` — include all 7 days, empty array if no slots checked for that day
    - Use slot values directly from `ARKBAR_SLOTS` (which already have en-dash characters) — never construct strings manually
    - PATCH to `/api/admin/djs/${djId}/recurring-availability` with `{ recurring_availability }`
    - On success: showToast, remove overlay, `loadManageDJs()`
    - On error: showToast error

    **3. `openFixedScheduleGrid(dj)` function — Fixed Schedule Modal:**

    Create a modal overlay (wider: `max-width: 800px`).

    Modal title: "Fixed Schedule: {dj.name}"

    Build **two sections** inside the modal:

    **Section A: ARKbar** (heading with teal accent)
    - Grid: 7 columns (Sun-Sat) x `ARKBAR_SLOTS.length` rows
    - Each cell: checkbox `id="fixed-arkbar-{dow}-{slotIndex}"`
    - Pre-load from `dj.fixed_schedules?.arkbar?.[String(dow)]` — check if slot is in the array

    **Section B: Love Beach Club** (heading with pink accent)
    - Grid: 7 columns (Sun-Sat) x rows
    - For Saturday (dow=6): use `LOVE_SAT_SLOTS`
    - For other days: use `LOVE_WEEKDAY_SLOTS`
    - Since the grid must be rectangular for a table, use `LOVE_SAT_SLOTS` (the superset, 9 slots) as rows for all days. For non-Saturday days, disable/grey-out checkboxes for slots that are NOT in `LOVE_WEEKDAY_SLOTS` (the extra Saturday-only slots: 17:00-18:00 and 18:00-19:00).
    - Each cell: checkbox `id="fixed-love-{dow}-{slotIndex}"`
    - Pre-load from `dj.fixed_schedules?.loveBeach?.[String(dow)]`

    Handle null `fixed_schedules` gracefully (all unchecked).

    Buttons: "Save" and "Cancel".

    **4. `saveFixedSchedule(djId)` function:**
    - Iterate ARKbar grid: for each dow, collect checked slot strings from `ARKBAR_SLOTS`
    - Iterate Love grid: for each dow, collect checked slot strings from `LOVE_SAT_SLOTS` (only enabled checkboxes)
    - Build `fixed_schedules` object:
      ```javascript
      {
        arkbar: { "0": [...], "1": [...], ... },
        loveBeach: { "0": [...], "1": [...], ... }
      }
      ```
    - Include all 7 days per venue, empty arrays for days with no slots
    - PATCH to `/api/admin/djs/${djId}/fixed-schedules` with `{ fixed_schedules }`
    - On success: showToast, remove overlay, `loadManageDJs()`
    - On error: showToast error

    **Important details:**
    - All slot values MUST come from the existing constant arrays (ARKBAR_SLOTS, LOVE_SAT_SLOTS, LOVE_WEEKDAY_SLOTS) — these already use en-dash U+2013
    - Never build slot strings by concatenation or from label text
    - Pre-loading must handle null/undefined JSONB columns (new DJs have null)
    - Grid scroll: wrap grids in `overflow-x: auto` div for mobile
