  Body: { pin }
  Returns: { success: true }

DELETE /api/admin/djs/:id/lockout
  Headers: x-admin-password
  Returns: { success: true }

<task type="auto">
  <name>Task 2: DJ table, Add DJ form, Edit modal, and action buttons</name>
  <files>public/roster.html</files>
  <action>
    Add the following JavaScript functions in roster.html's script section (after the switchVenue function area, before the data loading section). All functions follow the existing fetch + showToast pattern with `x-admin-password: adminPassword` header.

    **1. `loadManageDJs()` function:**
    - Fetch `GET /api/admin/djs` with admin header
    - On success: call `renderManageDJs(data.djs)`
    - On error: showToast with error message
    - Show `$('manageContent')` and set display to block
    - While loading, show spinner in manageContent (same pattern as loadAll)

    **2. `renderManageDJs(djs)` function:**
    - Build HTML into `$('manageContent')`:
      a. **Add DJ form** at top:
         - Inputs: Name (text, placeholder="DJ Name"), Rate (number, placeholder="Rate"), Type (select: resident/guest/casual), PIN (text, placeholder="PIN")
         - Submit button: "Add DJ"
         - Form calls `addNewDJ()` on submit

      b. **DJ table** with columns: Name, Rate, Type, Status, Lockout, Actions
         - For each DJ:
           - Name: plain text
           - Rate: integer display
           - Type: capitalize first letter
           - Status: `<span class="badge-active">Active</span>` or `<span class="badge-inactive">Inactive</span>`
           - Lockout: if `isLocked(dj)` show `<span class="badge-locked">Locked (N attempts)</span>`, else empty
           - Actions: Edit button, Deactivate/Reactivate button, Reset PIN button, Clear Lockout button (only if locked), Edit Availability button, Edit Fixed Schedule button
         - The last two buttons (availability, fixed schedule) will be wired in Plan 03 — for now render them as disabled buttons with `onclick="showToast('Coming soon','error')"` placeholder

    **3. `isLocked(dj)` helper:**
    - Returns `dj.locked_until && new Date(dj.locked_until) > new Date()`

    **4. `addNewDJ()` function:**
    - Read values from the add form inputs
    - POST to `/api/admin/djs` with `{ name, rate: parseInt(rate), type, pin }`
    - On success: showToast success, clear form, call `loadManageDJs()` to refresh table
    - On error: showToast error

    **5. `openEditDJModal(dj)` function:**
    - Create an overlay div with class `manage-edit-overlay`
    - Modal contains: Name input (pre-filled), Rate input (pre-filled), Type select (pre-filled)
    - Save button calls `saveEditDJ(dj.id)`, Cancel button removes overlay
    - Append overlay to document.body

    **6. `saveEditDJ(id)` function:**
    - Read values from modal inputs
    - PATCH to `/api/admin/djs/${id}` with `{ name, rate: parseInt(rate), type }`
    - On success: showToast, remove overlay, `loadManageDJs()`
    - On error: showToast error

    **7. `toggleDJActive(id, currentActive)` function:**
    - PATCH to `/api/admin/djs/${id}` with `{ active: !currentActive }`
    - On success: showToast, `loadManageDJs()`

    **8. `resetDJPin(id, name)` function:**
    - Use `prompt('Enter new PIN for ' + name + ':')` to get new PIN
    - If null/empty, return
    - POST to `/api/admin/djs/${id}/pin` with `{ pin }`
    - On success: showToast, no table refresh needed

    **9. `clearDJLockoutUI(id, name)` function:**
    - DELETE to `/api/admin/djs/${id}/lockout`
    - On success: showToast, `loadManageDJs()`

    **Important details:**
    - Store loaded DJs in a module-level variable `let manageDJs = [];` so Edit Availability/Fixed Schedule (Plan 03) can reference DJ data
    - All fetch calls use `headers: { 'x-admin-password': adminPassword }` (plus Content-Type for POST/PATCH)
    - All error handling: `if (!data.success) { showToast(data.error || 'Error', 'error'); return; }`
    - The manage tab should only be visible/functional when admin is authenticated (adminPassword is set). If adminPassword is empty and user clicks Manage DJs, show toast "Admin login required" and return.
