<objective>
Replace string equality password checks with bcrypt.compare for admin and manager passwords.

Purpose: Admin and manager passwords are currently compared using `===` against environment variables. This is both timing-unsafe (leaks info via timing side-channel) and means the passwords must be stored as plaintext in the environment. This plan changes all password comparisons to use bcrypt.compare against pre-hashed values stored in env vars.

Output: server.js with all admin/manager password checks using bcrypt.compare. Environment variables ADMIN_PASSWORD and MANAGER_PASSWORD will need to store bcrypt hashes instead of plaintext.
</objective>
