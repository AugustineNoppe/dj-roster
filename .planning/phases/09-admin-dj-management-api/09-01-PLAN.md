Create lib/admin-dj.js with a createAdminDJHandlers factory function following the exact pattern of lib/lockout.js.

Signature: `createAdminDJHandlers(supabase, bcrypt, invalidateCaches)`
- supabase: Supabase client instance (injected for testability)
- bcrypt: bcrypt module (injected for testability — allows mocking hash)
- invalidateCaches: callback function (injected — server.js passes its own invalidateCaches)

Returns: `{ listDJs, addDJ, editDJ, resetPin, clearLockout }`

Each function receives a plain object (not Express req/res) and returns a result object:
- listDJs() — no params needed
- addDJ({ name, rate, type, pin }) — returns { success, dj?, error?, status? }
- editDJ({ id, ...fields }) — returns { success, error?, status? }
- resetPin({ id, pin }) — returns { success, error?, status? }
- clearLockout({ id }) — returns { success, error?, status? }

Include `status` field (400, 500) in error returns so the route wiring in Plan 02 can set the HTTP status code. Default success has no status field (route defaults to 200).

For type validation: accept ['resident', 'guest', 'casual'] — this covers all known values. If the DB has a CHECK constraint that's narrower, the DB will catch it anyway, but server-side validation gives a better error message.

Use `'use strict';` at top. Export with `module.exports = { createAdminDJHandlers };`.
